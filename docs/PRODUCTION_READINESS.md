## 1. الملخص التنفيذي

«منارة» منصة SaaS متعددة المستأجرين لإدارة المدارس والمعاهد، بُنيت من نموذج Figma إلى منتج
حقيقي متعدد المنصات (API + ويب + ويندوز + موبايل) ثم خضعت لجولة تصليب مؤسسية من 6 محطات.

**الحالة:** النظام **جاهز لإطلاق تجريبي مُدار (Pilot/Beta)** مع مدرسة أو مدرستين حقيقيتين.
الوحدتان الأساسيتان — **الـ API والويب الإداري** — بمستوى إنتاجي: مصادقة محكمة، عزل مستأجرين
مُختبَر، تدقيق شامل، ترقيم سندات آمن، وتغطية اختبارية جيدة. تطبيق الموبايل (APK) وظيفي.
تطبيق ويندوز هيكله كامل لكن ملف التثبيت يتطلب سلسلة أدوات Rust غير مثبتة على جهاز التطوير.

**قبل التوسّع للإتاحة العامة (GA)** يجب إنجاز البنود الموثّقة في القسم 3 (نشر PostgreSQL،
بناء وتوقيع installer ويندوز، ربط مزوّدي SMS/WhatsApp/FCM الحقيقيين، وإعداد CI/CD).

**درجة الجاهزية: 84/100** (تفصيلها في القسم 10) · **مخاطر النشر: منخفضة–متوسطة** (القسم 11).

## 3. التوصيات المتبقية (قبل GA)

1. **نشر PostgreSQL:** بدّل provider في `prisma/schema.prisma` إلى `postgresql`، حوّل حقول
   String المعلَّمة `enum:` إلى enums، وشغّل `docker compose up`. (التطوير حاليًا SQLite.)
2. **installer ويندوز:** ثبّت Rust + VS Build Tools، `pnpm --filter @manarah/desktop build`،
   ثم وقّع الملف رقميًا (شهادة EV) لتفادي تحذير SmartScreen. (البنية جاهزة — انظر apps/desktop/README.)
3. **مزوّدو الإشعارات الحقيقيون:** استبدل `StubChannelProvider` بـ Twilio/WhatsApp Business،
   وأضف FCM للموبايل (النموذج والواجهة جاهزان).
4. **CI/CD:** أضف GitHub Actions يشغّل الاختبارات الـ48 + الفحوص على كل PR.
5. **وصولية:** ربط `htmlFor/id` لحقول الحوارات (Login مربوط؛ حوارات السند/الطالب مؤجَّلة).
6. **Swagger غني (L7):** جسر zod→OpenAPI لتوثيق DTOs كاملًا.
7. **إجبار تغيير كلمة المرور في الإنتاج:** `SEED_FORCE_PASSWORD_CHANGE=true` + سرّ JWT قوي.
8. **مراقبة:** أضف تسجيلًا مركزيًا (تنسيق موحّد موجود) + تنبيهات.

## 4. ملخص الأمان

- **المصادقة:** argon2 + JWT 15د + تدوير refresh بكشف إعادة الاستخدام وإبطال العائلة + قفل 5 محاولات. ✅ مُختبَر.
- **الصلاحيات (RBAC):** guards على مستوى الدور + `tenantWhere` للعزل + نطاق المعلم لشعبه + منع الحذف/الإلغاء إلا Super Admin. **صفر تصعيد صلاحيات معروف** بعد إصلاح H1/H2/M1.
- **عزل المستأجرين:** اختبار صريح يثبت رؤية صفر عبر المؤسسات. ✅
- **OWASP:** IDOR (مُصلَح H1/H2)، Broken Access Control (RBAC + isolation)، حقن (Prisma parameterized + zod)، رفع ملفات (MIME allow-list)، Rate limiting (global + auth مشدد)، Helmet + CSP، أسرار خارج الكود (`.env` غير مُتتبَّع). CSRF مخفَّف بـ SameSite=lax + Bearer.
- **الطباعة/التخزين:** مستندات print-ready بتهريب HTML، تخزين ملفات محلي مُحصَّن الاسم (S3-ready).
- **متبقٍ:** أسرار الإنتاج، httpOnly Secure cookies (secure=true عند NODE_ENV=production — مُهيأ).

**تقييم الأمان: قوي للوحدة الأساسية.** لا ثغرات Critical/High مفتوحة.

## 5. ملخص الأداء

- **الواجهة:** الحزمة الواحدة 1.26MB ← **34 chunk** (code-splitting)؛ index 308KB، recharts (518KB)
  خارج المسار الحرج، كل صفحة chunk مستقل 7–17KB. TanStack Query يخزّن ويُبطل بذكاء.
- **الـ API:** استعلامات مُفهرسة (فهارس tenantId/FK مُضافة)، تجميعات دفعة واحدة للملخصات،
  pagination server-side على الطلبة/الرسوم/التدقيق، سقوف أمان على القوائم الأخرى.
- **قابل للقياس:** الأداء الحقيقي تحت حمل يتطلب PostgreSQL + اختبار حمل (مؤجَّل، موثَّق).

**تقييم الأداء: جيد للنطاق المتوقع؛** يُعاد التحقق تحت Postgres قبل التوسّع.

## 6. ملخص قاعدة البيانات

- **المخطط:** 20 نموذجًا مشتقًا من الكيانات الحقيقية، مع unique constraints حرجة (ترقيم السندات
  `[tenantId, year, seq]`، الحضور `[studentId, date]`، النتائج `[examId, studentId]`).
- **السلامة:** ترقيم السندات داخل معاملة (بلا فراغات، مُختبَر)، soft-delete للطلبة، append-only للتدقيق.
- **الفهارس:** على tenantId والمفاتيح الأجنبية عبر النماذج كلها.
- **التطوير/الإنتاج:** SQLite للتطوير، PostgreSQL موثَّق للإنتاج (docker-compose جاهز).
- **متبقٍ:** المهاجرات الرسمية (`prisma migrate`) بدل `db push` عند اعتماد Postgres، وسياسات cascade.

**تقييم القاعدة: سليمة ومتّسقة** ضمن نطاق MVP.

## 7. ملخص المعمارية

Monorepo (pnpm) نظيف: `apps/api` (NestJS طبقي: controllers/services/guards/interceptors)،
`apps/web` (React + Router v7 + Query)، `apps/desktop` (Tauri 2)، `apps/mobile` (Flutter)،
`packages/shared` (مصدر الحقيقة للأنواع وzod)، `packages/api-client` (SDK موحّد بتجديد تلقائي).
فصل اهتمامات واضح، السيرفر هو حَكَم الصلاحيات، الواجهة تستهلك عقدًا واحدًا. **معمارية متينة وقابلة للتوسّع.**

## 8. ملخص جودة الكود

- **DRY:** توحيد `auditCtx`/`computeFeeStatus` (حذف 8 نسخ مكررة)، مكوّنات مشتركة (`QueryError` إلخ).
- **الأنواع:** TypeScript strict، صفر أخطاء tsc عبر الحزم، zod كمصدر تحقق موحّد بين السيرفر والواجهة.
- **صفر أزرار ميتة:** فحص آلي (88 معالج onClick) يمر بصفر مخالفات.
- **لا كود ميت:** لا مراجع mock.ts، لا TODO/FIXME حقيقية، لا استيرادات غير مستخدمة.
- **التعليقات:** عربية موجزة تشرح «لماذا» عند الحاجة.

**تقييم الجودة: عالٍ.**

## 12. الخلاصة النهائية

> **منصة «منارة» جاهزة لإطلاق تجريبي مُدار (Pilot) مع مؤسسة تعليمية حقيقية واحدة أو اثنتين
> على الويب والـ API.** الوحدتان الأساسيتان بمستوى إنتاجي مُختبَر، بصفر ثغرات حرجة أو عالية
> مفتوحة، وتغطية اختبارية عبر أربع طبقات (48 اختبارًا). الجولة المؤسسية كشفت وأصلحت 6 أخطاء
> High حقيقية — أربعة منها لم تظهر إلا بالتشغيل الحيّ و E2E — وهو دليل على عمق الفحص.
>
> **للإتاحة العامة (GA)** يلزم إنجاز بنود القسم 3 المحدودة والموثَّقة (نشر PostgreSQL، بناء
> وتوقيع installer ويندوز، ربط مزوّدي الإشعارات، وإعداد CI/CD). أيٌّ منها هندسي معروف المسار
> ولا يمسّ صحة النواة.
>
> **القرار: جاهز للإنتاج التجريبي المُدار الآن؛ جاهز للإتاحة العامة بعد بنود القسم 3.**
